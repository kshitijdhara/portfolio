SQL and NoSQL Databases Project

  • Tech Stack: MySQL, MongoDB, CosmosDB, Java, Hibernate, JUnit, Azure (ARM, CosmosDB)

Conducted an in-depth analysis and comparison of SQL and NoSQL databases, leveraging Yelp's Academic Dataset to gain practical insights into their respective strengths and use cases. Deployed and configured MySQL, a relational database management system (RDBMS), and MongoDB, a popular NoSQL document-oriented database, to store and query the dataset.

Utilized Azure Resource Manager (ARM) templates to streamline the provisioning and deployment of MySQL and CosmosDB (Azure's globally distributed, multi-model database service) instances on the Azure cloud platform. This approach ensured consistent and repeatable deployments across different environments.

Implemented data models and schemas tailored to the specific requirements of each database, taking into account factors such as data structure, relationships, and query patterns. Employed Java and Hibernate, an Object-Relational Mapping (ORM) framework, to interact with the databases and execute complex queries.

Conducted performance benchmarking and load testing to evaluate the scalability and responsiveness of the databases under various workloads. Implemented database indexing strategies and query optimization techniques to enhance query execution times and overall system performance.

Incorporated an in-memory key-value store to cache frequently accessed data, further improving response times and reducing the load on the primary databases. Utilized JUnit, a popular unit testing framework, to ensure the correctness and reliability of the implemented database operations and queries.

Throughout the project, adhered to best practices for database design, data modeling, and query optimization, ensuring data integrity, consistency, and efficient retrieval. Documented the findings and insights gained from the comparative analysis, providing valuable recommendations for selecting the appropriate database technology based on specific project requirements and data characteristics.
